# ESP32-NUT

ESP32-NUT is a downstream ESP32-S3 application built from Network UPS Tools
(NUT). It turns a supported ESP32-S3 board into a read-only UPS monitor with
USB HID polling, a LAN NUT server, and an authenticated HTTPS administration
console.

This is the application guide for the downstream project. The upstream NUT
source-tree overview remains in [README.adoc](README.adoc) for reference.

## What this application provides

- USB HID UPS monitoring through the ESP32-S3 native USB host
- CyberPower HID support, currently validated with a CyberPower CST150UC2
- Read-only NUT access on TCP `3493`
- LAN-only HTTPS administration on TCP `443`
- Per-device ADMIN authentication with secure sessions, CSRF protection, and
  failed-login throttling
- Named API tokens scoped to Agent OTA installation
- Wi-Fi provisioning, recovery, diagnostics, and device-owned time settings
- Authenticated local firmware checking and installation

TCP `8080` is a retired unauthenticated development service and must remain
closed. UPS control commands are not part of the current application; NUT
access remains read-only.

## Current operating model

The published operational baseline is `v2.5.0`. The current development slice,
branch, installed firmware, target observations, and exact next action are
maintained in [docs/ESP32_CURRENT_STATUS.md](docs/ESP32_CURRENT_STATUS.md).

The normal browser path uses Synology as a LAN reverse proxy:

```text
iPhone or Mac
    │ HTTPS with the trusted wildcard certificate
    ▼
Synology reverse proxy :443
    │ HTTPS to the ESP32; backend self-signed certificate accepted by policy
    ▼
ESP32-NUT :443
```

The application does not receive the wildcard private key. The complete
AdGuard/Synology procedure is in
[docs/ESP32_MANAGEMENT_PROXY.md](docs/ESP32_MANAGEMENT_PROXY.md).

## Supported target

The validated downstream target is:

- YD-ESP32-23
- ESP32-S3-WROOM-1-N16R8
- 16 MB flash with 8 MB octal PSRAM
- Native USB host connection to the UPS
- ESP-IDF v6.0.2

The retained initial partition layout uses the lower 8 MB of flash. Expanding
the layout is a separate change because dual-OTA capacity and recovery must be
preserved.

## Build

The supported local build environment is ESP-IDF v6.0.2:

```bash
. /Users/billyfkidney/.espressif/v6.0.2/esp-idf/export.sh
idf.py build
```

Generated ESP-IDF outputs such as `build/`, `managed_components/`, local
`sdkconfig` files, and `dependencies.lock` remain untracked. Flashing, OTA
installation, and physical recovery require the authority described in the
roles and preflight documents.

## Security boundaries

- Management is LAN-only HTTPS.
- Browser state-changing requests require the ADMIN session and CSRF header.
- API tokens are shown once, stored as verifiers, and scoped to Agent OTA.
- Wi-Fi setup is intentionally open only while provisioning or recovering;
  complete it from a trusted physical location.
- The HTTPS certificate generated by the current v2.x firmware is device
  specific and self-signed. The future local-CA replacement is a separate
  reviewed `v3.0.0` slice; the current reverse-proxy procedure does not copy
  the shared wildcard private key to a board.

Never record passwords, Wi-Fi credentials, session cookies, API tokens,
private keys, or Authorization headers in the repository or chat.

## Upstream relationship

ESP32-NUT preserves the upstream NUT layout, build inputs, documentation, and
driver architecture where practical. The downstream application layer adds
the ESP32-S3 hardware target, Wi-Fi lifecycle, HTTPS management, OTA behavior,
and read-only network service. Do not treat the upstream files as disposable
clutter; use this README and the linked downstream documents for application
operations.
